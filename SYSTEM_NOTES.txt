VOICE RECORD TRANSCRIPTOR - SYSTEM NOTES
=========================================

QUICK START
-----------
1. Set AUTH_TOKEN in .env file

2. Start services:
   docker compose up -d --build

3. Check logs:
   docker compose logs -f worker

4. Access UI:
   https://localhost:8443?token=your_secret_token_here

GPU CONFIGURATION
-----------------
GPU-only mode is enabled by default. CPU fallback is **not** used for Whisper.

Configure GPU via environment variables in docker-compose.yml:
- WHISPER_DEVICE=cuda (default: cuda, required for GPU-only mode)
- WHISPER_STRICT_CUDA=1 (default: 1)
  - When 1, any CUDA init error will cause jobs (and, on startup, the worker) to fail fast.
- WHISPER_COMPUTE_TYPE_CUDA=float16|int8_float16|... (default: float16)
- WHISPER_COMPUTE_TYPE_CUDA_FALLBACKS=int8_float16,int8 (tried on CUDA **only**, never CPU)
- WHISPER_COMPUTE_TYPE_CPU=int8|int8_float32|... (legacy / diagnostics only)

Legacy env vars are still understood for backwards compatibility (but CPU is not used for transcription):
- FORCE_DEVICE=cuda or cpu (default: cuda)
- CUDA_DEVICE_INDEX=0 (default: 0, use first GPU)
- CUDA_VISIBLE_DEVICES=0 (default: 0, expose GPU 0)
- COMPUTE_TYPE=float16 (default: float16 for GPU speed)

If you have multiple GPUs:
- Check available GPUs: docker compose exec worker nvidia-smi
- Set CUDA_DEVICE_INDEX to desired GPU index (0, 1, etc.)
- Set CUDA_VISIBLE_DEVICES to expose specific GPUs (e.g., "0,1")

SPEAKER DIARIZATION
-------------------
Speaker diarization uses fully local audio feature clustering:
- No external services or authentication required
- Uses MFCC features + K-means clustering
- Automatically detects 2-3 speakers by default
- Completely GPU/CPU local processing

The system analyzes voice characteristics (pitch, timbre, etc.) to identify
different speakers and labels them as SPEAKER_00, SPEAKER_01, etc.

VERIFY GPU USAGE
----------------
1. Check diagnostics endpoint:
   curl "http://localhost:8000/api/diagnostics?token=your_token"

2. Check selftest:
   curl "http://localhost:8000/api/selftest?token=your_token"

3. Check v2 health (GPU correctness):
   curl "http://localhost:8000/api/v2/health"
   # Look at: gpu_available, gpu_reason, selected_compute_type, strict_cuda

4. Watch worker logs during transcription:
   docker compose logs -f worker
   Look for lines like: "Model loaded OK on cuda:0 compute_type=float16"

5. Check nvidia-smi inside container:
   docker compose exec worker nvidia-smi

If you see errors like:
- "CUDA driver version is insufficient for CUDA runtime version"
it means the container CUDA runtime is newer than the host NVIDIA driver.

Fix options:
1. Upgrade NVIDIA driver on the host to match the container CUDA runtime.
2. Use a container image built with an older CUDA runtime compatible with the host driver.

If you see errors like:
- "libcuda.so: cannot open shared object file" or "no CUDA-capable device is detected"
it means the container is not getting GPU access.

Fix options:
1. Ensure Docker is started with GPU support (e.g. `docker run --gpus all` or compose device reservations).
2. Install and configure `nvidia-container-toolkit` on the host.

ARCHITECTURE
------------
- web: FastAPI server (no heavy processing, only queue management)
- worker: GPU transcription worker (reads from Redis queue)
- redis: Job queue (FIFO, concurrency=1 by default)
- proxy: Caddy reverse proxy with HTTPS

Upload flow:
1. Client uploads audio -> web service
2. web saves file to /data/sessions/SESSION_ID/audio.wav
3. web enqueues job to Redis
4. web returns immediately (non-blocking)
5. worker pops job from Redis
6. worker runs GPU transcription
7. worker saves results to /data/sessions/SESSION_ID/
8. Client polls /api/jobs/SESSION_ID for status

CONFIGURATION LIMITS
--------------------
MAX_UPLOAD_MB=200 (file size limit)
MAX_QUEUE=20 (max queued jobs before rejecting new uploads)
MAX_WORKERS=1 (transcription concurrency, keep at 1 for GPU)

TROUBLESHOOTING
---------------
Upload hangs:
- Check worker logs: docker compose logs worker
- Verify Redis: docker compose exec redis redis-cli ping
- Check queue: docker compose exec redis redis-cli llen transcription_jobs

GPU not used:
- Run diagnostics: curl localhost:8000/api/diagnostics?token=TOKEN
- Check CUDA_DEVICE_INDEX matches your GPU
- Verify nvidia-smi shows GPU inside container
- Check worker startup logs for smoke test result

Queue stuck:
- Clear queue: docker compose exec redis redis-cli del transcription_jobs
- Restart worker: docker compose restart worker
